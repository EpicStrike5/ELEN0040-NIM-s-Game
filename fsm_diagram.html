Transitions inside S_PLAY are evaluated in priority order every clk1 cycle. Only the highest-priority active condition fires — lower ones are skipped entirely. The coloured left border matches priority level (red = highest).
The synchronous reset rst = 1 overrides all other conditions from any state.

Variable Reference

x  (sticks_r)
unsigned(5:0)  ·  range 0–63
Number of sticks remaining on the table.
Set to rnd[4:0] + 9 (range 9–40) on START.
Decremented by sel_r on CONFIRM.
Modified ±Δ by JOKER2.
sel_r
unsigned(3:0)  ·  range 1–max_tk
Current player selection (how many sticks to take).
Incremented by UP, decremented by DOWN.
Reset to 1 after each CONFIRM turn.
Clamped down if JOKER1 reduces max_tk.
max_tk_r
unsigned(3:0)  ·  default 3
Maximum sticks a player may take per turn.
Default 3, reset to 3 on START.
Rerolled to rnd[2:0] + 2 (range 2–9) by JOKER1.
player_r
std_logic  ·  0=P1 / 1=P2
Whose turn it is.
Set from rnd[0] on START.
Toggled on every successful CONFIRM.
Also toggled before entering S_WIN — so player_r in S_WIN holds the winner.
rnd
std_logic_vector(6:0)  ·  range 0–127
Wire from sr_cnt in nim_sr.
Free-runs at clk0 (59–320 Hz) through all 128 values.
Sampled at the exact clk1 cycle a button fires — unpredictable to the player.
j1_p1 / j1_p2
j2_p1 / j2_p2
std_logic  ·  1=available / 0=spent
One flag per player per joker.
All four reset to 1 on START.
Cleared (0) when the active player uses that joker — each joker can only be used once per game per player.
